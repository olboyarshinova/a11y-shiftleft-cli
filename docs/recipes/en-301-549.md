# EN 301 549 Recipe

Use this recipe when a web project needs EN 301 549-oriented evidence context.
The CLI can help collect browser findings, WCAG mappings, screenshots,
keyboard evidence, and manual-review gaps, but it does not certify EN 301 549
or European Accessibility Act conformance.

ETSI describes EN 301 549 as a European standard for ICT products and services,
including web pages, mobile applications, desktop applications, hardware, and
combined hardware/software systems. This CLI preset is intentionally scoped to
web evidence only.

Official reference:

```txt
https://www.etsi.org/human-factors-accessibility/en-301-549-v3-the-harmonized-european-standard-for-ict-accessibility
```

## Install

```bash
npm install --save-dev a11y-shiftleft-cli
npx playwright install chromium
npx a11y-shiftleft init --framework auto
```

## Run

```bash
export APP_URL=http://localhost:5173
npx a11y-shiftleft doctor --url $APP_URL
npx a11y-shiftleft audit \
  --url $APP_URL \
  --standard en301549 \
  --out reports
```

Use the URL printed by your local dev server or preview environment.

## What This Covers

- WCAG-oriented automated findings for rendered web content.
- Visual screenshots and annotations for browser states the CLI safely reaches.
- Keyboard traversal evidence where enabled.
- Manual-review prompts for areas automation cannot prove.

## Explicit Gaps

This preset does not audit native mobile apps, desktop software, hardware,
telecommunications features, support services, procurement files, or
organizational documentation. Preserve separate manual evidence for those areas
when they are in scope.
